# ログイン・配布手順

## 方針

- QRコードはスマホ版・パソコン版で分けない。
- 共通URL `https://kiyori015.github.io/lingopilot/` を1つのQRコードにする。
- スマホではQRコードから開き、必要に応じてホーム画面に追加する。
- パソコンでは同じURLをブラウザで開き、必要に応じてブックマークする。
- 音声認識・音声再生の安定性を考え、参加者にはChrome系ブラウザを推奨する。

## 現在の公開状況（2026-07-12確認）

- 韓国語版: `https://kiyori015.github.io/lingopilot/`
- タイ語版: `https://kiyori015.github.io/lingopilot/thai.html`
- GitHub Pagesの両URLがHTTP 200で応答することを確認済み。
- `src/index.html` と `src/thai.html` のインラインJavaScript構文チェック済み。
- `lingopilot-auth` Edge Functionは外部から到達でき、無効ログインに対して `401 {"error":"invalid-login"}` を返すことを確認済み。

## 管理者の流れ

1. 管理者アカウントで管理画面にログインする。
2. ユーザー登録欄にメールアドレスを入力する。
3. 登録すると、登録順に `01`、`02`、`03`...の番号が発行される。
4. 画面に表示された案内文をメールで送る。
5. メールには登録番号、メールアドレス、アプリURL、QRコードを添える。

## 実利用開始チェックリスト

### 配布前

1. `git status` と `git diff --stat` で、意図しない未コミット変更がないか確認する。
2. `src/` に変更がある場合は `main` にpushし、GitHub Actionsの `Deploy GitHub Pages` 成功を確認する。
3. 管理者ログイン情報が外部共有されていないことを確認し、必要ならSupabase Authで管理者パスワードを変更する。
4. テスト用メールアドレスを1件登録し、登録番号+メールアドレスでログインできるか確認する。
5. テストユーザーで1問以上回答し、管理画面の記録一覧とCSVエクスポートに反映されるか確認する。
6. 研究利用の場合、参加者に伝える説明文、同意取得方法、問い合わせ先、利用期間を先に確定する。

### 配布時

1. 管理画面で参加者のメールアドレスを登録する。
2. 発行された登録番号を参加者ごとに控える。
3. 下の案内文テンプレートに、登録番号・メールアドレス・利用期間を入れて送る。
4. 参加者には、できるだけ同じ端末・同じブラウザで利用するよう伝える。
5. スマホ利用者には、初回ログイン後にホーム画面へ追加してもらう。

### 配布後

1. 初日のうちに管理画面で最終ログインと学習記録が入っているか確認する。
2. 記録が入らない参加者には、ログイン状態・通信状態・ブラウザを確認してもらう。
3. 定期的に管理画面の「全記録をCSVダウンロード」でバックアップを保存する。
4. 実験終了後はCSVを保存し、必要に応じて `profiles.status = 'inactive'` にして新規ログインを止める。

## ユーザーの流れ

1. 届いたQRコードまたはURLでアプリを開く。
2. トップ画面で登録番号とメールアドレスを入力する。
3. ログインできるとマイページが開く。
4. 2回目以降は同じ端末であればログイン状態が残る。

```text

以下のURLまたはQRコードからアプリを開いてください。
https://kiyori015.github.io/lingopilot/

登録番号: XX
メールアドレス: your-address@example.com

ログイン画面で、上の登録番号とメールアドレスを入力してください。
スマートフォンではChromeなどのブラウザで開くことをおすすめします。
2回目以降も学習記録をつなげるため、できるだけ同じ端末・同じブラウザで利用してください。

利用期間: YYYY年MM月DD日からYYYY年MM月DD日まで
困ったときの連絡先: （連絡先を入れる）
```

## Supabase側で必要な設定

- `supabase/migrations/202605310001_learning_records.sql` を適用する。
- `supabase/migrations/202606200001_registration_number_login.sql` を適用する。
- `lingopilot-auth` Edge Functionをデプロイする。
- Edge Functionに以下のシークレットを設定する。

```text
SUPABASE_SERVICE_ROLE_KEY
LINGOPILOT_LOGIN_PEPPER
LINGOPILOT_APP_URL=https://kiyori015.github.io/lingopilot/
```

`LINGOPILOT_LOGIN_PEPPER` は任意の長いランダム文字列にする。ユーザーには共有しない。

## CLIで本番反映する場合

以下の情報が必要。

- `SUPABASE_ACCESS_TOKEN`
- `SUPABASE_DB_PASSWORD` または `SUPABASE_DB_URL`
- 必要に応じて `SUPABASE_SERVICE_ROLE_KEY`

PowerShellで一時的に環境変数を設定してから実行する。

```powershell
$env:SUPABASE_ACCESS_TOKEN = "SupabaseのAccess Token"
$env:SUPABASE_DB_PASSWORD = "SupabaseのDB Password"
.\scripts\deploy-supabase-production.ps1
```

`LINGOPILOT_LOGIN_PEPPER` は未設定ならスクリプト実行時に自動生成される。

## 注意

- 登録番号とメールアドレスを知っている人はログインできるため、案内メールを第三者に転送しないように伝える。
- 半年程度のお試し運用ならこの方式で扱いやすい。
- より厳密な本人確認が必要になった場合は、メールOTPまたはMagic Linkに戻す。
