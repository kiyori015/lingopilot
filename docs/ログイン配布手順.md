# ログイン・配布手順

## 方針

- QRコードはスマホ版・パソコン版で分けない。
- 共通URL `https://kiyori015.github.io/lingopilot/` を1つのQRコードにする。
- スマホではQRコードから開き、必要に応じてホーム画面に追加する。
- パソコンでは同じURLをブラウザで開き、必要に応じてブックマークする。

## 管理者の流れ

1. 管理者アカウントで管理画面にログインする。
2. ユーザー登録欄にメールアドレスを入力する。
3. 登録すると、登録順に `01`、`02`、`03`...の番号が発行される。
4. 画面に表示された案内文をメールで送る。
5. メールには登録番号、メールアドレス、アプリURL、QRコードを添える。

## ユーザーの流れ

1. 届いたQRコードまたはURLでアプリを開く。
2. トップ画面で登録番号とメールアドレスを入力する。
3. ログインできるとマイページが開く。
4. 2回目以降は同じ端末であればログイン状態が残る。

## 配布QRコード

- PNG: `src/assets/lingopilot-app-qr.png`
- SVG: `src/assets/lingopilot-app-qr.svg`
- URL: `https://kiyori015.github.io/lingopilot/`

## Supabase側で必要な設定

- `supabase/migrations/202605310001_learning_records.sql` を適用する。
- `supabase/migrations/202606200001_registration_number_login.sql` を適用する。
- `lingopilot-auth` Edge Functionをデプロイする。
- Edge Functionに以下のシークレットを設定する。

```text
SUPABASE_SERVICE_ROLE_KEY
LINGOPILOT_LOGIN_PEPPER
LINGOPILOT_APP_URL=https://kiyori015.github.io/lingopilot/
```

`LINGOPILOT_LOGIN_PEPPER` は任意の長いランダム文字列にする。ユーザーには共有しない。

## CLIで本番反映する場合

以下の情報が必要。

- `SUPABASE_ACCESS_TOKEN`
- `SUPABASE_DB_PASSWORD` または `SUPABASE_DB_URL`
- 必要に応じて `SUPABASE_SERVICE_ROLE_KEY`

PowerShellで一時的に環境変数を設定してから実行する。

```powershell
$env:SUPABASE_ACCESS_TOKEN = "SupabaseのAccess Token"
$env:SUPABASE_DB_PASSWORD = "SupabaseのDB Password"
.\scripts\deploy-supabase-production.ps1
```

`LINGOPILOT_LOGIN_PEPPER` は未設定ならスクリプト実行時に自動生成される。

## 注意

- 登録番号とメールアドレスを知っている人はログインできるため、案内メールを第三者に転送しないように伝える。
- 半年程度のお試し運用ならこの方式で扱いやすい。
- より厳密な本人確認が必要になった場合は、メールOTPまたはMagic Linkに戻す。
