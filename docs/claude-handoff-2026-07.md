# 引き継ぎ書: Claude Code による改修まとめ（2026年7月）

このドキュメントは、2026/07/04〜07/07 に Claude Code で行ったアプリ改修の引き継ぎ書である。
発表資料・論文の作成や今後の開発時に、このファイルを一次資料として参照すること。
対応するコミット範囲: `110b964`〜`05afede`（各コミットメッセージにも詳細あり）。

## 1. アプリの現在の全体像

- 韓国語版 `src/index.html`・タイ語版 `src/thai.html` の2つの単一ファイルSPA（バニラJS、フレームワークなし）
- 公開: GitHub Pages https://kiyori015.github.io/lingopilot/ （`main` に push すると `.github/workflows/pages.yml` が `src/` を自動デプロイ。所要約20秒）
- 認証・データ同期: Supabase（プロジェクト `ieakvpwzhihqttcxegti`、**リージョンは ap-south-1**）
  - ログイン: 登録番号+メールアドレス → Edge Function `lingopilot-auth` → Supabase Auth
  - 学習データ: `user_app_state` / `daily_learning_records`（タイ語版は `thai_` プレフィックス）、`profiles` は共通
- 主な学習フロー: 章選択 → ステップ（1.場面 → 2.聞く → 3.基本 → 4.文法 → 5.単語 → 6.カード → 7.聞き取り → 8.練習 → 9.結果 → 10.話す）

## 2. 今回の改修内容（発表・論文で言及できる機能）

### 2.1 フィードバックの視認性強化（`110b964`）
- クイズ選択肢: 正解=緑背景+✓、誤答=赤背景+✕、結果文「⭕ 正解！」「❌ 不正解…」を色付き太字で表示
- 単語カード: 正誤で枠・背景色が変わり「○ 正解」「× 不正解」バッジ表示。押したボタンは塗りつぶし表示
- 文法説明の図解化: 「名詞＋이에요/예요 → 丁寧な『〜です』」形式の文型ブロック図+例文中の該当箇所ハイライト
  - 文法データが「日本語ラベル: 現地語表現」形式の場合、現地語側をブロックにするパーサー（`parseGrammarPoint`）

### 2.2 復習リマインド（`110b964`）
- 間隔反復: 章完了後、翌日→3日後→7日後に復習を提案（`REVIEW_INTERVAL_DAYS`）
- 押し忘れ対策: ①画面左下の🔔固定アラート ②ナビの「復習n」バッジ ③復習セッション中に8回操作すると**自動で復習完了を記録**（`REVIEW_AUTO_COMPLETE_ACTIONS`）

### 2.3 話す練習（音声認識）の改善（`110b964`）
- 認識結果にカナ読み+**日本語訳・英語訳**を表示（MyMemory API、後述の制限あり）
- 満点条件の明示: 「キーワードを2つ以上で3点」+キーワードチップが認識されると緑✓に変化
- 言い方の例を3つ表示（基本/別の言い方/困ったとき）。ミッションデータに `alternatives` フィールド追加

### 2.4 学習記録の堅牢化（`6989cab`）— 論文の実験データ収集の根幹
- ログイン時の読込失敗で同期が永久停止するバグを修正（保存のたびに再送）
- リモート読込時、日別記録を**クリアせず日別maxマージ**（端末にしかない記録を保護）
- 学習中は**1分ごとに学習時間を確定して即時同期**（クラッシュしても最大1分の損失）
- タブ非表示・ページ終了・ログアウト時に未送信分を即時フラッシュ
- タイ語版管理画面にもSupabase全ユーザー記録一覧を実装（韓国語版から移植）

### 2.5 CSVエクスポート（`ea37eed`）
- ユーザー: ホーム「学習記録をファイルに保存（CSV）」→ 日付・時間・解答数・正答率・完了章（BOM付きExcel対応）
- 管理者: 「全記録をCSVダウンロード」→ 全ユーザー×全日付（登録番号・日付順）。**論文のデータ分析にそのまま使える**

### 2.6 ユーザビリティ改善 P1（`27dc5cf`）
- 端末に韓国語/タイ語TTS音声がない場合、ホームに赤い警告を自動表示（`updateVoiceSupportNotice`）
- ログイン画面に推奨ブラウザ（Chrome系）を明記
- 「データ保存の状態」カードを折りたたみ化、開始ボタンを強調
- ステップタブに番号付与、「聞く」重複を「聞き取り」に改名

### 2.7 ユーザビリティ改善 P2（`14ea578`）
- 2分無操作時: 画面はそのままで**学習時間の計測のみ一時停止**（操作で自動再開）
- 問題量削減: 4択 各方向14→10問、リスニング12→8問
- **「もう一度挑戦（新しい問題）」**（`restartChapterQuiz`）: 誤答した表現を優先的に多めに出題し、
  出題順・選択肢・問題文の言い回し（3パターン）が毎回変わる
- 韓国語版の問題生成をシード方式に変更（`quizSeed`+`createSeededRng`）。
  従来は再読込のたびに問題と採点記録がズレる潜在バグがあった（タイ語版は元からシード方式）

### 2.8 スマホ最適化（`782e231`）
- 4択・リスニング・単語カードを**1問/1枚ずつ表示**に変更（即時採点+1.1秒後自動送り）
- 一括採点ボタン（章の採点/答え合わせ）は廃止（hidden）
- ステップタブはモバイルで横スクロール1行化、ヘッダーの装飾チップ等を非表示

### 2.9 結果画面の残項目表示（`05afede`）
- 判定待ちのとき「のこり: 韓→日 あと3問・聞き取り あと8問」と具体的に表示（採点済み項目は出さない）

## 3. 運用・インフラの知識

- **マイグレーション適用**: `pwsh ./scripts/apply-migrations.ps1`
  - 直結ホスト `db.*.supabase.co` はIPv6のみで接続不可。**プーラー `aws-1-ap-south-1.pooler.supabase.com:5432`** を使う（スクリプトに設定済み）
  - 適用済み: `202607050001_last_login_at.sql` まで全て適用済み（2026/07/06 ダッシュボードSQL Editorで実施）
- **公開**: `git push origin main` だけでよい（Actionsが自動デプロイ）
- **秘密情報**: `.env.local`（DBパスワード、サービスロールキー、ログインペッパー）。コミット禁止
- `scripts/deploy-supabase-production.ps1` は `SUPABASE_ACCESS_TOKEN` が必要で、**ペッパー未設定時に再生成してログインを壊す危険がある**ため、DBだけなら `apply-migrations.ps1` を使うこと

## 4. 既知の制限・注意点（論文の限界の節にも使える）

1. **MyMemory翻訳API**: 「話す」の認識結果翻訳のみで消費（1回の認識=2リクエスト）。匿名は約5,000文字/日/IP。
   端末にキャッシュ（最大300件、1件180文字まで）。上限超過時も読み方・採点は動作する
2. **音声認識はChrome系のみ**（iOS SafariのSpeechRecognitionは不安定）。TTSは端末の音声データに依存
3. **話す採点はキーワード包含判定のみ**で発音の質は評価していない
4. **PWAキャッシュ**: 更新後、既に開いているタブは再読み込みが必要（SWはネットワーク優先なのでリロードで最新化）
5. **コード重複**: 韓国語版とタイ語版でロジックの大部分が重複（計約1.9万行）。修正は必ず両ファイルに適用すること
6. リマインドの**メール送信は未実装**（アプリ内通知のみ）。実装するならSupabase cron+メールAPIが必要

## 5. 開発時の便利な手法（Claudeセッションで確立したもの）

- **ローカル動作確認**: `python -m http.server 4173`（`src/` を配信）または `.claude/launch.json` の設定
- **ログインを通さずテスト**: ブラウザのコンソールで
  ```js
  remoteSyncEnabled = false; applyAuthenticatedState("zztest", false); signedInAuthUserId = "";
  ```
  （テスト後は `localStorage` の `zztest` を含むキーを削除）
- **インラインJSの構文チェック**: `<script>` 中身を抽出して `new Function(code)` に通す（Node）
- **修正は必ず両ファイル**: 韓国語版 `speakSentence/speakSlow`・タイ語版 `speakText/speakSlow`、
  韓国語版 `chapter.quizSets`・タイ語版 `state.exerciseSets` のように関数名・データ持ち方が微妙に違う点に注意

## 6. ⚠️ 最重要の注意

**同じ修正依頼を Claude と Codex に同時に投げないこと。**
2026/07/04 に同一プロンプトが両方で並行実行され、同じファイルに同時書き込みが発生した（幸い衝突は回避）。
どちらか一方で作業し、終わってから次のツールに切り替えること。作業開始前に `git status` と
`git diff --stat` で予期しない変更がないか確認する習慣を推奨。
