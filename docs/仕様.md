# 言語学習アプリ 仕様（韓国語・初級）

## 概要
- 対象: 韓国語の全くの初心者
- 目的: 日常会話でよく使う表現を、短い会話と反復練習で身につける
- 形式: マイページ + テキスト学習 + 単語帳
- 学習の流れ: まずマイページで学習記録を確認し、そこから続きのテキスト学習または単語学習へ進む

## 画面構成

### 1. マイページ
- アプリ起動時に最初に表示する
- 一番上に以下の導線ボタンを配置する
  - `学習を始める`
  - `単語学習を始める`
- 続きから学べるように、現在の進捗章を表示する
- 以下の学習記録を表示する
  - 今日の学習時間
  - 連続学習日数
  - 累計学習時間
  - 累計解答数
  - 累計正答率
  - 完了章数
  - 得意分野 / 苦手分野
  - 月間カレンダー形式の学習記録
  - 直近7日の日別サマリー
  - 連続学習バッジ

### 2. テキスト
- マイページの `学習を始める` または `続きから始める` から遷移する
- 毎回最初からではなく、進めたところから再開できる
- 章一覧から任意の章を選べる
- 章ごとに以下を表示する
  - 基本例文
  - 文法ポイント
  - 会話例文
  - 単語リスト
  - 単語カード
  - 四択問題
  - 一部章のリスニング問題
  - 学習結果と補講導線

### 3. 単語帳
- マイページの `単語学習を始める` から遷移する
- 全章の単語をまとめて見られる
- 章別フィルタを用意する
- 単語カード形式で反復学習する

## 学習記録機能
- カレンダー形式で日ごとの学習記録を表示する
- 日ごとに以下を集計する
  - 学習時間
  - 解答数
  - 正答率
  - 完了章数
- 連続学習日数を表示する
- 連続学習に応じて以下のバッジを付与する
  - 3日連続
  - 1週間連続
  - 2週間連続
  - 1ヶ月連続
- 章別の正答率傾向から、得意分野・苦手分野を表示する

## 採点・復習ロジック
- 採点対象
  - 四択問題
  - 単語カード
  - リスニング問題
- 判定基準
  - `○`: 80%以上
  - `△`: 60〜79%
  - `×`: 59%以下
- 復習方針
  - `△`: 文章問題メインで復習する
  - `×`: 単語メインで復習する
- 補講では、誤答問題を同じ内容で使い回さず、基本文法や重要語彙を保ちながら別パターンの問題に差し替える
- 四択の選択肢は重複しないように生成する

## 出題方針
- 章ごとの基本文法と重要語彙は固定する
- ただし、問題文は語句や組み合わせを変えながら再生成する
- 同じ章を再挑戦するときに、常にまったく同じ文章にならないようにする
- 例文は「韓国語 / 日本語訳」で表示する
- 音声再生と録音は、基本例文と会話例文で利用できる

## 単語帳仕様
- 全章の単語を横断して復習できる
- 章ごとの絞り込みができる
- 表面に韓国語、裏面に日本語を表示する
- 自己採点結果は学習記録に反映する

## 保存仕様
- 進捗と学習記録の正の保存先はSupabase Postgresにする
- ブラウザのローカルストレージは、通信不良時・未同期時の一時保存として使う
- 保存対象
  - 現在の章
  - 章ごとの解答状況
  - 単語カードの自己採点
  - 学習履歴
- Supabaseに保存する主なテーブル
  - `profiles`: ユーザーのメールアドレス、表示名、権限、利用状態
  - `user_app_state`: 現在の章、章ごとの進捗、単語カード、リマインド、質問メモ
  - `daily_learning_records`: 日別の学習時間、解答数、正答数、完了章数
- ユーザー本人は自分の記録だけ確認できる
- 管理者は全ユーザーの記録を確認できる
- GitHubはコード・仕様・DBマイグレーションの管理に使い、学習記録そのものは保存しない

## ログイン・利用開始仕様
- 一般公開はしない
- 管理者が管理画面でメールアドレスを登録する
- メールアドレスを登録した順に `01`、`02`、`03`...の登録番号を発行する
- 管理者は登録番号、メールアドレス、アプリURL、QRコードをユーザーに送る
- 初回利用時はQRコードでアプリURLを開き、登録番号とメールアドレスでログインする
- 2回目以降はSupabase Authのセッションを利用し、アプリアイコンまたはブックマークから開けるようにする
- スマホではPWAとしてホーム画面に追加できる形を目指す
- QRコードはスマホ版・パソコン版で分けず、共通URL `https://kiyori015.github.io/lingopilot/` を使う

## 表記ルール
- 例文は「韓国語 / 日本語訳」を併記する
- 文体は丁寧体（〜요）を基本にする
- 初級者向けに短く、日常ですぐ使える形を優先する
